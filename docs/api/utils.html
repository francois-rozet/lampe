General purpose helpers.

lampe.utils.cache(f=None, /, persist=False)#

Unbounded function cache decorator.

Wraps a function with a memoizing callable that saves call results, which can save time when an expensive function is called several times with the same arguments.

The positional and keyword arguments of f must be hashable.

Parameters

  • f (Optional[Callable]) – The function to decorate.

  • persist (bool) – Whether the cached values persist to disk or not.

Example

>>> @cache
... def fib(n):
...     return n if n < 2 else fib(n-2) + fib(n-1)
...
>>> fib(42)
267914296
lampe.utils.vectorize(f=None, /, **kwargs)#

Convenience vectorization decorator.

Defines a vectorized function which takes a sequence of objects or NumPy arrays as inputs and returns a tuple of NumPy arrays. The vectorized function evaluates f over successive tuples of the input arrays like the map function, except it uses the broadcasting rules of NumPy.

Parameters

  • f (Optional[Callable]) – The function to decorate.

  • kwargs – Keyword arguments passed to numpy.vectorize.

Example

>>> @vectorize(otypes=[float])
... def abs(x):
...     return x if x > 0 else -x
...
>>> abs(range(-3, 4))
array([3., 2., 1., 0., 1., 2., 3.])
lampe.utils.broadcast(*tensors, ignore=None)#

Broadcasts tensors together.

The term broadcasting describes how PyTorch treats tensors with different shapes during arithmetic operations. In short, if possible, dimensions that have different sizes are expanded (without making copies) to be compatible.

Parameters

ignore (Optional[Union[int, List[int]]]) – The number(s) of dimensions not to broadcast.

Example

>>> x = torch.rand(3, 1, 2)
>>> y = torch.rand(4, 5)
>>> x, y = broadcast(x, y, ignore=1)
>>> x.shape
torch.Size([3, 4, 2])
>>> y.shape
torch.Size([3, 4, 5])
lampe.utils.starcompose(*fs)#

Returns the composition \(g\) of a sequence of functions \((f_1, f_2, \dots, f_n)\).

\[g = f_n \circ \dots \circ f_2 \circ f_1\]

If the output \(x_i\) of the intermediate function \(f_i\) is a tuple, its elements are used as separate arguments for the next function \(f_{i+1}\).

Parameters

fs (Callable) – A sequence of functions \((f_1, f_2, \dots, f_n)\).

Returns

The composition \(g\).

Return type

Callable

Example

>>> g = starcompose(lambda x: x**2, lambda x: x/2)
>>> g(5)
12.5
class lampe.utils.GDStep(optimizer, clip=None)#

Creates a callable that performs gradient descent (GD) optimization steps for parameters \(\phi\) with respect to differentiable loss values.

The callable takes a scalar loss \(l\) as input, performs a step

\[\phi \gets \text{GD}(\phi, \nabla_{\!\phi} \, l)\]

and returns the loss, detached from the computational graph. To prevent invalid parameters, steps are skipped if not-a-number (NaN) or infinite values are found in the gradient. This feature requires CPU-GPU synchronization, which could be a bottleneck for some applications.

Parameters

  • optimizer (Optimizer) – An optimizer instance (e.g. torch.optim.SGD).

  • clip (float) – The norm at which the gradients are clipped. If None, gradients are not clipped.

class lampe.utils.PlateauDetector(threshold=0.01, patience=8, mode='min')#

Creates a plateau detector for online sequences.

Each time a new value \(x_t\) is provided, it is compared with the current best value \(x_b\) to determine whether \(t\) is the new best time step. In the minimization mode, \(x_t\) is considered better if it satisfies

\[x_t < x_b \, (1 - \tau) ,\]

where \(\tau \in [0, 1]\) is a significance threshold. If it is the case, \(b\) becomes \(t\). The sequence is currently at a plateau if \(b\) has not changed for more than \(\lambda\) patience steps, i.e. if

\[t - b > \lambda .\]
Parameters

  • threshold (float) – The significance threshold \(\tau\).

  • patience (int) – The patience \(\lambda\).

  • mode (str) – The improvement mode, either 'min' or 'max'.

class lampe.utils.MetropolisHastings(x_0, f=None, log_f=None, sigma=1.0)#

Creates a batched Metropolis-Hastings sampler.

Metropolis-Hastings is a Markov chain Monte Carlo (MCMC) sampling algorithm used to sample from intractable distributions \(p(x)\) whose density is proportial to a tracatble function \(f(x)\), with \(x \in \mathcal{X}\). The algorithm consists in repeating the following routine for \(t = 1\) to \(T\), where \(x_0\) is the initial sample and \(q(x' | x)\) is a pre-defined transition distribution.

  1. sample \(x' \sim q(x' | x_{t-1})\)

  2. \(\displaystyle \alpha \gets \frac{f(x')}{f(x_{t-1})} \frac{q(x_{t-1} | x')}{q(x' | x_{t-1})}\)

  3. sample \(u \sim \mathcal{U}(0, 1)\)

  4. \(x_t \gets \begin{cases} x' & \text{if } u \leq \alpha \\ x_{t-1} & \text{otherwise} \end{cases}\)

Asymptotically, i.e. when \(T \to \infty\), the distribution of samples \(x_t\) is guaranteed to converge towards \(p(x)\). In this implementation, a Gaussian transition \(q(x' | x) = \mathcal{N}(x'; x, \Sigma)\) is used, which can be modified by subclassing MetropolisHastings.

Wikipedia

https://en.wikipedia.org/wiki/Metropolis%E2%80%93Hastings_algorithm

Parameters

  • x_0 (Tensor) – A batch of initial points \(x_0\), with shape \((*, L)\).

  • f (Callable) – A function \(f(x)\) proportional to a density function \(p(x)\).

  • log_f (Callable) – The logarithm \(\log f(x)\) of a function proportional to \(p(x)\).

  • sigma (Union[float, Tensor]) – The standard deviation of the Gaussian transition. Either a scalar or a vector.

Example

>>> x_0 = torch.rand(128, 7)
>>> log_f = lambda x: -(x**2).sum(dim=-1) / 2
>>> sampler = MetropolisHastings(x_0, log_f=log_f, sigma=0.5)
>>> samples = [x for x in sampler(2**8, burn=2**7, step=2**2)]
>>> samples = torch.stack(samples)
>>> samples.shape
torch.Size([32, 128, 7])
lampe.utils.gridapply(self, f, bins, bounds, batch_size=4096)#

Evaluates a function \(f(x)\) over a multi-dimensional domain split into grid cells. Instead of evaluating the function cell by cell, batches are given to the function.

Parameters

  • f (Callable) – A function \(f(x)\).

  • bins (Union[int, List[int]]) – The number(s) of bins per dimension.

  • bounds (Tuple[Tensor, Tensor]) – A tuple of lower and upper domain bounds.

  • batch_size (int) – The size of the batches given to the function.

Example

>>> f = lambda x: -(x**2).sum(dim=-1) / 2
>>> lower, upper = torch.zeros(3), torch.ones(3)
>>> y = gridapply(f, 50, bounds=(lower, upper))
>>> y.shape
torch.Size([50, 50, 50])
