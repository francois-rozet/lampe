lampe.data#

Datasets and data loaders.

lampe.data.JointLoader(prior, simulator, batch_size=1024, batched=False, numpy=False, **kwargs)#

Creates a data loader of batched pairs \((\theta, x)\) generated by a prior distribution \(p(\theta)\) and a simulator.

The simlator is a stochastic function taking a set of parameters \(\theta\) as input and returning an observation \(x\) as output, which (implicitely) defines a likelihood function \(p(x | \theta)\). Together with the prior, they form a joint distribution \(p(\theta, x) = p(\theta) p(x | \theta)\) from which the pairs \((\theta, x)\) are independently drawn.

Parameters

  • prior (Distribution) – A prior distribution \(p(\theta)\).

  • simulator (Callable) – A callable simulator.

  • batch_size (int) – The batch size of the generated pairs.

  • batched (bool) – Whether the simulator accepts batched inputs or not.

  • numpy (bool) – Whether the simulator requires NumPy or PyTorch inputs.

  • kwargs – Keyword arguments passed to torch.utils.data.DataLoader.

Returns

An infinite data loader of batched pairs \((\theta, x)\).

Return type

DataLoader

Example

>>> loader = joint_loader(prior, simulator, numpy=True, num_workers=4)
>>> for theta, x in loader:
...     theta, x = theta.cuda(), x.cuda()
...     something(theta, x)
class lampe.data.H5Dataset(*files, batch_size=1024, chunk_size=4096, group_size=16, pin_memory=False, shuffle=True, seed=None)#

Creates a dataset of pairs \((\theta, x)\) from HDF5 files.

As a torch.utils.data.Dataset, H5Dataset implements the methods __len__ and __getitem__. However, as it can be slow to load pairs from disk one by one when iterating over the dataset, it also implements a custom __iter__ method. This method loads several contiguous chunks of pairs at once, concatenates them, shuffles the result and, finally, splits it into batches. This “weak shuffling” procedure greatly improves loading performances, but the resulting batch elements are not perfectly independent from each others.

Important

To take advantage of the custom __iter__ method, H5Dataset instances should not be wrapped inside a torch.utils.data.DataLoader when iterating over the dataset.

Parameters

  • files (Union[str, Path]) – HDF5 files containing pairs \((\theta, x)\).

  • batch_size (int) – The size of the batches.

  • chunk_size (int) – The size of the contiguous chunks.

  • group_size (str) – The number of chunks loaded at once.

  • pin_memory (bool) – Whether the batches reside in CUDA pinned memory or not.

  • shuffle (bool) – Whether the pairs are shuffled when iterating.

  • seed (int) – A seed to initialize the internal RNG used for shuffling.

Example

>>> dataset = H5Dataset('data.h5', batch_size=256, shuffle=True)
>>> theta, x = dataset[0]
>>> theta
tensor([-0.1215, -1.3641,  0.7233, -1.2150, -1.9263])
>>> for theta, x in dataset:
...     theta, x = theta.cuda(), x.cuda()
...     something(theta, x)
static store(pairs, file, size, dtype=<class 'numpy.float32'>, **meta)#

Creates an HDF5 file containing pairs \((\theta, x)\).

The sets of parameters \(\theta\) are stored in a collection named 'theta' and the observations in a collection named 'x'.

Parameters

  • pairs (Iterable[Tuple[ndarray, ndarray]]) – An iterable over batched pairs \((\theta, x)\).

  • file (Union[str, Path]) – An HDF5 filename to store pairs in.

  • size (int) – The number of pairs to store.

  • dtype (dtype) – The data type to store pairs in.

  • meta – Metadata to store in the file.

Example

>>> loader = JointLoader(prior, simulator, batch_size=16)
>>> H5Dataset.store(loader, 'sim.h5', 4096)
100%|██████████| 4096/4096 [01:35<00:00, 42.69sample/s]
